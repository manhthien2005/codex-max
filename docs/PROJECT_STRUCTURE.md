# Project Structure

This document describes the WSL-first working layout of the Codex workspace.

---

## Top Level

```text
.
├── AGENTS.md
├── config.toml
├── hooks.json
├── README.md
├── README.vi.md
├── .agents/
├── agents/
├── docs/
├── hooks/

├── mcp_template/     ← published MCP launcher/source templates
├── rules/
├── scripts/
├── skills/
└── vendor_imports/
```

## Skills Surfaces

### Curated source library

`skills/` remains the source of truth for:

- curated skill content,
- `CATALOG.md`,
- `manifest.yaml`,
- maintainer-only workflows,
- internal `.system` support skills.

### Runtime discovery surface

`.agents/skills/` is the repo-local skill surface that Codex should discover at runtime.

It is generated by [`scripts/sync-runtime-skills.sh`](../scripts/sync-runtime-skills.sh) as a symlinked mirror of the curated library.

User-scope discovery lives at:

```text
$HOME/.agents/skills -> ~/.codex/.agents/skills
```

## Hooks

`hooks/` now targets WSL/Linux as the real runtime surface.

Main files:

```text
hooks/
├── codex_hook_adapter.py
├── hook-probe.py
├── post_tool_use.py
├── pre_tool_use.py
├── rtk-shell-init.sh
├── session-start.sh
├── stop.py
├── user-prompt-submit.sh
└── user_prompt_submit.py
```

Key behavior:

- `SessionStart` creates the diary sentinel and runs planning catch-up if available.
- `UserPromptSubmit` now emits JSON `hookSpecificOutput.additionalContext` through `user_prompt_submit.py`, while the shell script remains a legacy/manual helper.
- `PreToolUse` and `PostToolUse` only apply to Bash.
- `Stop` remains the diary gate.

## MCP Surfaces

### Published source

`mcp_template/` is the git-tracked source of truth for local MCP launchers and adapter code.

### Local runtime

`mcp/` is machine-local and gitignored. It contains:

- Linux virtual environments,
- copied launchers from `mcp_template/`,
- local caches and Python entrypoints.

Bootstrap it with:

```bash
bash ~/.codex/scripts/bootstrap-mcp-wsl.sh
```

## Rules

`rules/default.rules` is the active WSL-oriented rule surface.

The previous Windows-heavy allowlist is archived under:

```text
rules/archive/windows-default.rules
```

## Local-Only Runtime State

These remain intentionally untracked:

- `mcp/`
- caches and model downloads
- sessions and SQLite files
- `.tmp/`
- auth and machine-local state

See [docs/INSTALLATION.md](INSTALLATION.md) for setup and verification.
